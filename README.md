# 微型制钟师

静态浏览器小游戏，直接打开 `index.html` 即可运行。

玩家根据故障记录选择零件，将零件拖到钟面对应插槽，或点击零件快速装配。完成后调节摆长和齿轮咬合，运行走时测试观察误差趋势，根据多轮采样结果进行校准，最终获得误差与评分。

## 游戏模式

### 自由修理模式

默认模式，提供 3 个预设委托，零件库存无限。玩家可以自由切换委托，反复测试直到满意为止。每个委托会记录最佳成绩。

### 每日工坊挑战模式

点击顶部「每日工坊挑战」按钮切换到此模式。

**核心特性：**

- **每日更新**：同一天打开页面时，始终生成同一组 3 个委托和相同的零件库存。基于日期种子生成，保证可复现。
- **有限库存**：每个零件只有 1-3 件可用，需要合理规划零件使用。更换已装配的零件会将旧零件退回库存。
- **顺序挑战**：必须按顺序完成委托，当前委托验收通过后才能解锁下一个。
- **评分系统**：每个委托根据误差获得 S/A/B/C/D 评级，对应 100/80/60/40/20 分。完成全部 3 个委托后显示总分。
- **进度持久化**：挑战进度、库存状态、测试记录都保存在本地存储，刷新页面不会丢失。次日自动重置为新的挑战。

## 操作说明

1. 从左侧零件托盘点击或拖拽零件到钟面对应的插槽
2. 根据故障记录和验收目标选择合适的零件组合
3. 调节摆长（每格 -3 秒/日）和咬合（每格 +2.2 秒/日）滑块进行微调
4. 点击「走时测试」按钮检测实际走时误差（含 ±2.5 秒随机波动）
5. 查看评分和验收结果，根据需要调整零件或微调

**挑战模式额外操作：**

- 挑战委托列表会显示当前进度、已完成委托的成绩
- 零件按钮上显示剩余库存，库存不足的零件会被禁用
- 侧边栏显示今日零件库存实时变化
- 完成所有委托后显示今日总评分

## 测试

误差计算和评分规则已抽出为纯计算模块 `calc.js`（`window.CalcEngine` / CommonJS 双模式），不依赖 DOM 与随机扰动，可稳定测试。覆盖用例：

- 目标零件全匹配（零零件误差）
- 缺失零件罚分（擒纵 42 / 其他 34）
- 错误零件逐槽差值
- 摆长微调 −3/格、咬合微调 +2.2/格
- 零件+微调综合场景
- 指定 jitter 的带扰动结果
- 评分档位 S/A/B/C/D 全部边界
- 验收判定与 gap
- 缺省参数、未知零件等防御性场景

### 方式一：浏览器打开（零依赖）

打开 `tests/test.html`，页面会自执行所有用例并以分组形式展示通过/失败。

### 方式二：命令行 Node.js（零依赖）

```bash
node tests/calc.test.js
```

退出码 0 表示全部通过，非 0 表示有失败用例并打印错误详情。

## 技术实现

### 多轮校准测试系统

走时测试采用多轮采样校准机制，提供可解释的误差趋势：

1. **种子化随机**：基于零件配置+委托+测试次数生成种子，保证相同配置结果可复现
2. **委托特性**：每个委托有独特的漂移（drift）和波动率（volatility）
3. **6 次采样**：每次测试分 6 个采样点，包含预热偏差和逐渐稳定的过程
4. **趋势分析**：通过线性回归计算误差斜率，判断走时趋势方向
5. **统计指标**：提供均值、标准差、极差、趋势等丰富数据

### 每日挑战数据生成

使用线性同余生成器（LCG）实现基于日期的可复现伪随机序列：

1. 日期字符串（如 "2024-06-15"）通过 hash 函数转换为整数种子
2. 使用种子初始化伪随机数生成器
3. 从 12 个预设委托模板中随机选择 3 个
4. 为每个委托随机生成目标零件组合和验收目标
5. 为每种零件生成 1-3 件的初始库存

### 本地存储键

- `clockmaker_mode` - 当前游戏模式
- `clockmaker_daily_challenge` - 今日挑战数据（委托、库存种子）
- `clockmaker_inventory` - 当前库存状态
- `clockmaker_challenge_index` - 当前挑战委托索引
- `clockmaker_challenge_results` - 挑战委托完成结果
- `clockmaker_free_best` - 自由模式最佳成绩
- `clockmaker_free_history` - 自由模式测试历史

## 文件结构

```
├── calc.js              # 纯计算：误差/评分/验收（已抽离，可被测试稳定覆盖）
├── data.js              # 订单与零件数据 + 每日挑战生成器
├── store.js             # 全局状态（含模式切换、库存、进度持久化）
├── diagnostics.js       # 诊断提示（复用 calc.js）
├── manual.js            # 修理手册面板（复用 calc.js）
├── dom.js / events.js   # DOM 查询与事件绑定（含模式切换事件）
├── renderer.js / app.js # 渲染与启动（含挑战 UI 渲染）
├── styles.css           # 样式（含挑战模式 UI）
├── index.html           # 页面结构（含模式切换、挑战面板）
└── tests
    ├── calc.test.js     # Node.js 端测试（内置 assert，无额外依赖）
    └── test.html        # 浏览器端可视化测试
```
